## 本次保留的改动

### 1. 后端请求 worker panic 保护
- 文件：`crates/service/src/http/backend_runtime.rs`
- 文件：`crates/service/src/http/tests/backend_runtime_tests.rs`
- 保留原因：
  - 单个请求处理 panic 时不再直接打死 worker。
  - 日志会记录 panic 的 method、path 和 payload，便于排查。

### 2. 用量轮询分批与周期预算
- 文件：`crates/service/src/usage/usage_refresh.rs`
- 文件：`crates/service/src/usage/tests/usage_refresh_tests.rs`
- 保留原因：
  - 新增 `CODEXMANAGER_USAGE_POLL_BATCH_LIMIT`
  - 新增 `CODEXMANAGER_USAGE_POLL_CYCLE_BUDGET_SECS`
  - 大账号量场景下不再每轮全量扫描，避免单次轮询时间过长。
  - 通过游标推进让下一轮从上次位置继续，整体更平滑。

### 3. 后台循环 panic 后自动拉起
- 文件：`crates/service/src/usage/usage_refresh.rs`
- 保留原因：
  - `usage-polling`、`gateway-keepalive`、`token-refresh-polling` 改为统一后台线程包装。
  - 后台线程异常时记录日志并重启，降低“线程静默退出”风险。

### 4. 日期展示 bug 修复
- 文件：`apps/src/utils/format.js`
- 文件：`apps/src/utils/tests/format.test.js`
- 保留原因：
  - 修复 `formatResetLabel()` 中可能出现的 `月月` 展示问题。
  - 已补对应前端测试。

### 5. Web 端 RPC 地址回环修正
- 文件：`apps/src/api.js`
- 保留原因：
  - 当地址是 `0.0.0.0:port` 或 `127.0.0.1:port` 时，Web/RPC 层统一走 `localhost:port`。
  - 这部分语义清晰，能避免浏览器侧直接拿监听地址作为连接地址。

### 6. README 中已实现配置项补充
- 文件：`README.md`
- 保留原因：
  - 保留已真实实现的 `CODEXMANAGER_USAGE_POLL_BATCH_LIMIT`
  - 保留已真实实现的 `CODEXMANAGER_USAGE_POLL_CYCLE_BUDGET_SECS`

## 本次移除的改动

### 1. 未实现的 watchdog 文档
- 文件：`README.md`
- 移除内容：
  - `CODEXMANAGER_WATCHDOG_ENABLED`
  - `CODEXMANAGER_WATCHDOG_INTERVAL_SECS`
  - `CODEXMANAGER_WATCHDOG_FAILURE_THRESHOLD`
  - `CODEXMANAGER_WATCHDOG_RESTART_COOLDOWN_SECS`
- 移除原因：
  - 当前仓库代码中没有对应功能实现。
  - 直接保留会把未实现能力写进正式文档，误导用户。

### 2. 根目录额外说明文档
- 文件：`KEY配置使用说明.md`
- 移除原因：
  - 该文件不是本次 PR 主线的一部分。
  - 放在仓库根目录会增加维护噪音。
  - 后续如需要对外文档，应统一进入 `docs/` 或 README 体系。

### 3. 前端连接层“保留 0.0.0.0”改法
- 文件：`apps/src/services/connection.js`
- 文件：`apps/src/services/tests/connection.test.js`
- 移除原因：
  - PR 中把 `normalizeAddr()` 改成保留 `0.0.0.0`，同时又引入 `normalizeRpcAddr()` 做回环修正。
  - 这会让“监听地址”和“连接地址”概念在前端层混杂。
  - 当前主干已有更清晰的做法：对连接入口直接归一成 `localhost`，由后端负责 bind 地址转换。

### 4. 启动遮罩展示 `0.0.0.0`
- 文件：`apps/src/ui/startup-mask.js`
- 移除原因：
  - `0.0.0.0` 是 bind addr，不适合直接展示给用户。
  - 启动提示应继续显示实际连接目标，如 `localhost:48760`。

### 5. 与本次主题无关的样式改动
- 文件：`apps/src/styles/base.css`
- 文件：`apps/src/styles/components/part-2.css`
- 移除原因：
  - 这部分属于额外视觉微调。
  - 与“后台任务与网关增强”主线无强关联，不应跟着后端稳定性改动一起进入主干。
